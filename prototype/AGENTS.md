# AGENTS.md

# SCOPE

This folder owns the web prototype for Design Copilot MVP.

Default stack:

- Next.js
- TypeScript
- UI-only where possible
- mock data before APIs

Prototype the current MVP modules only:

1. Prompt Builder
2. Reference Analyzer
3. Variation Generator

Out of scope:

- production plugin implementation
- full Adobe automation
- backend-heavy architecture
- one-click final design generation

# UX RULES

Target working designers.

The UI should feel:

- fast
- minimal
- practical
- production-oriented

Every screen should answer:

"How does this save designer time?"

# WORKING RULES

Plan first.
Prototype second.
Code third.

Prefer:

- small reviewable diffs
- modular components
- mock flows that designers can test
- clear concept direction outputs

Challenge complexity before adding it.

# ACCEPTANCE

The prototype is useful only if a real designer can test whether it reduces concept-start friction.
