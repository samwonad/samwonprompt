# Commercial Content Intent Taxonomy

# Purpose

Design Copilot should support any commercial banner or sign content, not only events.

The engine should infer the commercial intent from the user's rough brief and generate concept prompts around that intent.

# Supported Content Intents

## 1. Event / Campaign

Examples:

- Children's Day event
- Parents' Day event
- seasonal campaign
- pop-up event

Design priority:

- date clarity
- event name visibility
- target audience separation
- festive impact

## 2. Sale / Discount

Examples:

- clearance sale
- limited-time discount
- one plus one
- price promotion

Design priority:

- offer readability
- price or discount scale
- urgency
- fast street recognition

## 3. Menu / Product Push

Examples:

- new menu
- best seller
- signature product
- lunch set

Design priority:

- product appetite or desirability
- product name hierarchy
- price or benefit clarity
- visual recognition

## 4. Opening / Relocation

Examples:

- grand opening
- reopening
- store relocation
- new branch

Design priority:

- business name
- location
- opening message
- trust and discovery

## 5. Service Information

Examples:

- clinic service
- academy program
- repair service
- consulting service

Design priority:

- service category clarity
- benefit hierarchy
- contact/action readability
- credibility

## 6. Recruitment / Notice

Examples:

- staff wanted
- class registration
- reservation notice
- operating hours

Design priority:

- notice purpose
- required action
- conditions
- contact or schedule

## 7. Brand / Storefront Recognition

Examples:

- facade signboard
- store identity banner
- brand introduction
- category recognition

Design priority:

- business name visibility
- category recognition
- facade compatibility
- distance readability

# MVP Rule

Do not add a separate UI field for every intent yet.

For v0.3, use one freeform brief field and infer intent internally from keywords.
Only add more fields after designer testing proves that inference is not enough.
