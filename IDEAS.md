# IDEAS.md

# Purpose

Capture product ideas for Design Copilot MVP without letting scope expand too early.

Every idea should be judged by one question:

"Does this help designers reach usable starting concepts faster?"

# MVP Idea Candidates

## Prompt Builder

- Structured image2 prompt builder using layers:
  - subject
  - style
  - composition
  - variation directives
  - production notes
- Brief-to-prompt form for banner and sign projects.
- Copy field for required text, headline, offer, dates, and location.
- Prompt preview that designers can quickly edit before generation.

## Reference Analyzer

- 2-3 reference upload slots.
- Manual reference notes before automated analysis.
- Extract reusable style directions:
  - typography hierarchy
  - layout structure
  - color/contrast direction
  - visual impact
  - commercial readability
- Convert reference analysis into prompt-ready language.

## Variation Generator

- Generate A/B/C concept directions from one brief.
- Force each variation to differ by composition strategy, not only color.
- Label variations by intent:
  - typography-led
  - image-led
  - high-contrast minimal
- Provide designer-facing rationale for each direction.

# Later Ideas

These are not MVP unless validated by designer testing.

- Illustrator plugin.
- Adobe automation.
- Brand kit import.
- Saved project history.
- Team prompt library.
- Automatic export presets.
- One-click layout reconstruction.
- Advanced image scoring.
- Signboard facade mockup mode.
- Banner hierarchy presets.
- Korean retail promo prompt sets.

# Rejected Or Risky Ideas

Avoid unless there is strong evidence:

- Fully automatic final artwork generation.
- Complex backend orchestration before UI validation.
- Generic poster engine.
- Broad consumer design app features.
- Too many prompt modes.

# Idea Evaluation Checklist

Use this before promoting an idea into the prototype:

1. Does it reduce design-start friction?
2. Does it improve variation exploration?
3. Can a real designer test it quickly?
4. Can it work with mock data first?
5. Is it inside the current MVP scope?
